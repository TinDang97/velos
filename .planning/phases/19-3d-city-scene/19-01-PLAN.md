  <action>
Create `crates/velos-net/src/building_import.rs` following the `osm_import.rs` two-pass pattern:

1. Define `BuildingFootprint` struct with `polygon: Vec<[f64; 2]>` (exterior ring in projected metres) and `height_m: f64`.

2. Implement `compute_building_height(tags)` function:
   - Check `height` tag first (parse, handle " m" suffix)
   - Then `building:levels` tag (parse * 3.5)
   - Default 10.5m (3 floors)

3. Implement `ensure_ccw(polygon)` using signed area (shoelace formula). If negative (CW), reverse.

4. Implement `pub fn import_buildings(pbf_path: &Path, proj: &EquirectangularProjection) -> Result<Vec<BuildingFootprint>, NetError>`:
   - Pass 1: Collect all node id->coord pairs using `ElementReader::new` + `Element::Node`/`Element::DenseNode`
   - Pass 2: Extract `Element::Way` with any `building=*` tag. Resolve node refs to projected coords via `proj.project()`. Compute height. Ensure CCW winding. Skip ways with < 3 unique vertices.
   - Only Way-type buildings for POC (not multipolygon Relations -- document this limitation)

5. Update `crates/velos-net/src/lib.rs`: add `pub mod building_import;` and `pub use building_import::{BuildingFootprint, import_buildings};`

Use the same `osmpbf` crate patterns as `osm_import.rs`. Reuse `EquirectangularProjection` for coordinate conversion. The PBF file is `data/hcmc/district1.osm.pbf` (same file used for roads).
  </action>

  </verify>

  <name>Task 2: Building extrusion geometry and shader</name>
  <files>crates/velos-gpu/src/building_geometry.rs, crates/velos-gpu/src/lib.rs, crates/velos-gpu/shaders/building_3d.wgsl</files>
  <behavior>
    - Test: BuildingVertex is 40 bytes (position [f32;3] + normal [f32;3] + color [f32;4])
    - Test: BuildingVertex derives Pod + Zeroable
    - Test: generate_building_geometry with a simple square footprint produces correct vertex count (4 walls * 4 verts + 2 roof tris + 2 base tris worth of indexed geometry)
    - Test: wall normals point outward (perpendicular to wall face)
    - Test: roof normals are [0, 1, 0] (pointing up)
    - Test: building_color_with_variation produces RGBA values within base color +/-5%
    - Test: building_3d.wgsl passes naga validation
  </behavior>
  <action>
Create `crates/velos-gpu/src/building_geometry.rs`:

1. Define `BuildingVertex` with `#[repr(C)]`, `Pod`, `Zeroable`:
   - `position: [f32; 3]` -- world position
   - `normal: [f32; 3]` -- face normal for lighting
   - `color: [f32; 4]` -- per-vertex color (base beige with variation)
   Total: 40 bytes per vertex.

2. Implement `building_color_with_variation(centroid_x: f64, centroid_y: f64) -> [f32; 4]`:
   - Base color #D4C5A9 = (0.831, 0.773, 0.663)
   - Hash centroid for deterministic +/-5% brightness variation
   - Return RGBA with alpha 1.0

3. Implement `pub fn generate_building_geometry(buildings: &[BuildingFootprint]) -> (Vec<BuildingVertex>, Vec<u32>)`:
   - For each building:
     a. Compute centroid, derive per-building color
     b. Triangulate roof using `earcutr::earcut()` -- vertices at Y=height, normal=[0,1,0]
     c. Generate wall quads between consecutive polygon vertices:
        - 4 vertices per wall segment (bottom-left, bottom-right, top-right, top-left)
        - Base at Y=0.0, top at Y=height
        - Outward normal: `cross(up, edge_direction)` for CCW polygon
        - Two indexed triangles per quad
     d. 2D (x, y) maps to 3D (x, Y, y) per established convention
   - Return single merged vertex + index buffer for all buildings (single draw call)

4. Create `crates/velos-gpu/shaders/building_3d.wgsl`:
   - Same CameraUniform and LightingUniform structs as mesh_3d.wgsl
   - @group(0) @binding(0) camera, @group(0) @binding(1) lighting
   - VertexInput: position vec3<f32> @location(0), normal vec3<f32> @location(1), color vec4<f32> @location(2)
   - NO instancing (buildings are not instances -- each has unique geometry in merged buffer)
   - vs_main: transform position by view_proj, pass normal and color to fragment
   - fs_main: Lambert diffuse + ambient shading (same as mesh_3d.wgsl fragment shader)

5. Update `crates/velos-gpu/src/lib.rs`: add `pub mod building_geometry;`

Note: Do NOT add instancing to the building shader -- buildings are unique geometry in a single buffer, unlike agents which are identical meshes instanced. The vertex buffer contains per-vertex color already baked in.
  </action>
