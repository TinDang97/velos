    - Test: terrain color is muted green #3a5a3a = (0.227, 0.353, 0.227, 1.0)
  </behavior>
  <action>
Create `crates/velos-gpu/src/terrain.rs`:

1. Define `TerrainVertex` with `#[repr(C)]`, `Pod`, `Zeroable`:
   - `position: [f32; 3]` -- world position with elevation as Y
   - `color: [f32; 4]` -- uniform muted green
   Total: 28 bytes (same layout as ground_plane vertex).

2. Constants:
   - `SRTM1_SAMPLES: usize = 3601`
   - `SRTM3_SAMPLES: usize = 1201`
   - `SRTM_VOID: i16 = -32768`
   - `GROUND_COLOR: [f32; 4] = [0.227, 0.353, 0.227, 1.0]` -- #3a5a3a
   - `MAX_TERRAIN_Y: f32 = -0.5` -- clamp below road level

3. Implement `pub fn parse_hgt(path: &Path) -> Result<(Vec<i16>, usize), io::Error>`:
   - Read file bytes
   - Determine resolution from file size (SRTM1 vs SRTM3)
   - Parse as flat Vec<i16> (big-endian), row-major order (north to south)
   - Replace void values (-32768) with 0 (sea level)
   - Return (elevation_data, samples_per_side)

4. Implement `pub fn generate_terrain_mesh(elevations: &[i16], samples: usize, tile_lat: f64, tile_lon: f64, proj: &EquirectangularProjection, bbox: Option<(f64, f64, f64, f64)>) -> (Vec<TerrainVertex>, Vec<u32>)`:
   - If bbox provided (min_lat, min_lon, max_lat, max_lon), clip grid to only vertices within the bounding box. This avoids generating the full 3601x3601 mesh.
   - For each grid point within bbox:
     a. Compute lat/lon from grid row/col (lat = tile_lat + 1.0 - row * step, lon = tile_lon + col * step)
     b. Project to world coords via proj.project(lat, lon) -> (x, y_proj)
     c. Elevation: if void (already replaced with 0), use 0.0m. Scale elevation to world units. HCMC is flat (0-15m), so elevation in metres is the Y value.
     d. Clamp Y to MAX_TERRAIN_Y (-0.5) to stay below road surface
     e. 2D projected (x, y_proj) maps to 3D (x, Y, y_proj)
   - Generate indexed triangles: for each (row, col) quad, two triangles (tl-bl-tr, tr-bl-br)
   - Return (vertices, indices) for single draw call

5. Update `crates/velos-gpu/src/lib.rs`: add `pub mod terrain;`

The bbox should come from the road graph extent at runtime (Plan 03 will pass it). For unit tests, use synthetic small grids.

Manual .hgt parsing is preferred over a crate -- the format is trivially simple (raw big-endian i16 grid). No new dependencies needed.

Create `crates/velos-gpu/shaders/terrain.wgsl` following the `ground_plane.wgsl` pattern exactly:

3. VertexInput: @location(0) position vec3<f32>, @location(1) color vec4<f32> (same as ground_plane)
