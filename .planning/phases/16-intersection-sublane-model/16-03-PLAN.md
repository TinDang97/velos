**B) Create `crates/velos-gpu/shaders/map_tile.wgsl`:**

Simple colored polygon shader that transforms tile vertices by the camera uniform:

```wgsl
// map_tile.wgsl -- Colored polygon rendering for map tiles.
// Shares camera uniform layout with agent shader.

struct CameraUniform {
    view_proj: mat4x4<f32>,
};

@group(0) @binding(0) var<uniform> camera: CameraUniform;

struct VertexInput {
    @location(0) position: vec2<f32>,
    @location(1) color: vec4<f32>,
};

struct VertexOutput {
    @builtin(position) clip_position: vec4<f32>,
    @location(0) color: vec4<f32>,
};

@vertex
fn vs_main(in: VertexInput) -> VertexOutput {
    var out: VertexOutput;
    out.clip_position = camera.view_proj * vec4<f32>(in.position, 0.0, 1.0);
    out.color = in.color;
    return out;
}

@fragment
fn fs_main(in: VertexOutput) -> @location(0) vec4<f32> {
    return in.color;
}
```

The shader uses position (vec2<f32>) + color (vec4<f32>) per vertex. The camera uniform is the same layout as the agent shader (shared bind group).

**2. MapTileRenderer struct:**
- `pmtiles: Option<pmtiles2::PMTiles<std::io::BufReader<std::fs::File>>>` -- opened from local .pmtiles file path
- `tile_cache: LruCache<(u8, u64, u64), GpuTile>` -- capacity 128 per discretion recommendation
- `decoded_rx: mpsc::Receiver<DecodedTile>` -- receives from background thread
- `request_tx: mpsc::Sender<(u64, u64, u8)>` -- sends to background thread
- `pending: HashSet<(u8, u64, u64)>` -- tiles currently being decoded (avoid duplicate requests)
- `pipeline: wgpu::RenderPipeline` -- map_tile.wgsl pipeline
- `camera_bind_group: wgpu::BindGroup` -- shared with Renderer or duplicated

**3. Constructor `MapTileRenderer::new(device, format, camera_bind_group_layout, pmtiles_path: Option<&Path>)`:**
- If pmtiles_path is Some and file exists: open with `pmtiles2::PMTiles::from_reader(BufReader::new(File::open(path)))`
- Spawn background decode thread with the PMTiles reader clone (or use mpsc channels for requests)
- Create render pipeline from map_tile.wgsl
- Initialize empty LRU cache

**4. Background thread logic:**
The background thread owns a clone of the PMTiles reader (or receives tile read requests via channel):
- Receive `(x, y, z)` request
- Call `pmtiles.get_tile(x, y, z)` to get raw tile bytes
- Decompress with `flate2::read::GzDecoder` if gzip-compressed
- Decode MVT with `mvt_reader::Reader::new(&decompressed_bytes)`
- For each layer (buildings, water, landuse, roads, parks):
  - Get features via `reader.get_features(layer_idx)`
  - For polygon features: triangulate with `earcutr::earcut(&coords, &holes, 2)`
  - For line features: generate thin quads (2 triangles per segment, 1m width)
  - Assign color per layer per RESEARCH.md discretion:
    - Buildings: [0.20, 0.18, 0.22, 0.8]
    - Water: [0.15, 0.25, 0.40, 0.9]
    - Roads: [0.25, 0.25, 0.30, 0.6]
    - Parks/green: [0.15, 0.25, 0.15, 0.7]
    - Land use: [0.18, 0.17, 0.20, 0.5]
  - Reproject each vertex from MVT tile-local coordinates to Web Mercator to VELOS local metres using the projection module. MVT coords are in tile-local extent (typically 4096), convert to Web Mercator first, then use velos_net::projection::lonlat_to_local() (but convert from Mercator metres to lon/lat first).
  - IMPORTANT (RESEARCH.md Pitfall 3): PMTiles/MVT use Web Mercator (EPSG:3857). VELOS uses local metres projected from WGS84. Reproject each coordinate using the same projection origin (10.7756, 106.7019).
- Send DecodedTile via channel

**5. `fn update(&mut self, camera: &Camera2D, device: &wgpu::Device)`:**
- Receive any decoded tiles from background thread, create GPU buffers on main thread (RESEARCH.md Pitfall 4: buffer creation must happen on main thread)
- Compute visible tile coords via `visible_tiles(camera, zoom_level)`
- Zoom level mapping per discretion: camera.zoom < 0.5 -> z14, 0.5-2.0 -> z15, 2.0-8.0 -> z16, >8.0 -> z16
- For each visible tile not in cache and not pending: send request to background thread
- LRU touch for visible tiles (keeps them from eviction)

**6. `fn visible_tiles(camera: &Camera2D, zoom: u8) -> Vec<(u64, u64, u8)>`:**
- Convert camera viewport bounds from local metres back to lon/lat
- Convert lon/lat to tile x/y at given zoom level using standard slippy map math: `x = floor((lon + 180) / 360 * 2^z)`, `y = floor((1 - ln(tan(lat_rad) + sec(lat_rad)) / pi) / 2 * 2^z)`
- Return all tiles in the rectangular range

**7. `fn render(&self, encoder: &mut wgpu::CommandEncoder, view: &wgpu::TextureView, camera_bind_group: &wgpu::BindGroup)`:**
- Begin render pass (load existing, no clear -- tiles render before agents)
- For each tile in cache that is visible: draw vertex_buffer with vertex_count
- End render pass
