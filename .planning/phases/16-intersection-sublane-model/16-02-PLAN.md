  <name>Task 1: Junction traversal logic in velos-vehicle</name>
  <files>crates/velos-vehicle/src/junction_traversal.rs, crates/velos-vehicle/src/lib.rs</files>
  <behavior>
    - advance_on_bezier: given t=0.0, speed=10m/s, arc_length=20m, dt=0.1 -> new_t ~= 0.05
    - advance_on_bezier: t clamped to max 1.0, returns finished=true when t >= 1.0
    - check_conflicts: two agents on crossing turns near conflict t-values -> returns virtual leader distance
    - check_conflicts: agent past conflict point -> no conflict (foe_t > conflict_t + margin)
    - check_conflicts: agents on non-crossing turns -> no conflict
    - yield_deceleration: uses IDM with virtual leader at conflict crossing point distance
    - yield_deceleration: priority agent (closer to conflict point) does not decelerate
    - Tie-breaking: Emergency > Truck/Bus > Car > Motorbike when |t - t_cross| is equal
  </behavior>
  <action>
Create `crates/velos-vehicle/src/junction_traversal.rs` with:

1. `fn advance_on_bezier(t: f64, speed: f64, arc_length: f64, dt: f64) -> (f64, bool)`:
   - Compute `dt_param = dt * speed / arc_length.max(1.0)`
   - Return `(new_t.min(1.0), new_t >= 1.0)`
   - This gives approximately uniform speed along the curve (see RESEARCH.md Pitfall 2)

2. `struct ConflictCheckResult { pub virtual_leader_gap: f64, pub virtual_leader_speed: f64 }`:
   - Returned when an agent must yield to a foe at a conflict point

3. `fn check_conflicts(own_turn_idx: u16, own_t: f64, own_type: VehicleType, agents_in_junction: &[(u16, f64, VehicleType)], conflicts: &[ConflictPoint], t_proximity: f64) -> Option<ConflictCheckResult>`:
   - For each conflict involving own_turn_idx, find if a foe agent is on the other turn
   - Check t-proximity: both |own_t - conflict_t| < t_proximity AND |foe_t - foe_conflict_t| < t_proximity
   - Skip if foe has already passed conflict point (foe_t > foe_conflict_t + t_proximity)
   - Priority: agent closer to crossing point (lower |t - t_cross|) has priority
   - Tie-breaking: use size_factor ordering from existing intersection.rs (Emergency=5 > Truck=4 > Bus=3 > Car=2 > Motorbike=1)
   - If yielding: compute virtual leader gap as distance along curve from own_t to conflict_t (proportional to arc_length)
   - Use t_proximity = 0.15 as default (tunable, ~3m on a 20m curve per RESEARCH.md Pitfall 5)

4. `fn yield_deceleration(own_speed: f64, virtual_leader_gap: f64, virtual_leader_speed: f64, idm: &IdmParams) -> f64`:
   - Treat conflict crossing point as virtual leader
   - Apply IDM car-following: `idm.acceleration(own_speed, own_speed - virtual_leader_speed, virtual_leader_gap)`
   - Clamp to [-9.0, idm.a_max] per architecture rules

5. `fn size_factor(vtype: VehicleType) -> u8` -- priority ordering for tie-breaking

Add `pub mod junction_traversal;` to `crates/velos-vehicle/src/lib.rs`.

IMPORTANT: Per user decision, lateral offset is LOCKED when agent enters junction -- no dynamic lateral filtering inside junction areas. Motorbikes still filter freely on approach and departure edges (existing sublane.rs logic handles this, no changes needed).

**A) Create `crates/velos-gpu/src/sim_junction.rs`** with `step_junction_traversal(&mut self, dt: f64)`:

Per RESEARCH.md code example:
- Query all entities with `&JunctionTraversal` component
- For each: look up `JunctionData` from `self.junction_data` (HashMap<u32, JunctionData>)
- Call `advance_on_bezier(jt.t, jt.speed, turn.arc_length, dt)` to get new t
- Collect all agents in same junction for conflict checking: build Vec<(turn_idx, t, VehicleType)> per junction_node
- Call `check_conflicts()` for each agent; if yielding, call `yield_deceleration()` and apply deceleration to speed
- Compute world position via `turn.offset_position(new_t, jt.lateral_offset, road_half_width)` where road_half_width = entry_edge.lane_count * 3.5 / 2.0
- Compute heading from `turn.tangent(new_t)` as `atan2(tan[1], tan[0])`
- Update Position, Kinematics components
- If finished (t >= 1.0): remove JunctionTraversal, call advance_to_next_edge with overflow = 0.0 to place agent on exit edge

**B) Modify `crates/velos-gpu/src/sim_helpers.rs`** `advance_to_next_edge()`:

Before the current edge transition logic, intercept to check if the target node has a junction:
- Get target node from route (the node the agent is arriving at)
- If `self.junction_data.contains_key(&target_node_u32)`:
  - Find the BezierTurn matching (current_edge -> next_edge) in junction.turns
  - Lock the agent's current lateral_offset (read from LateralOffset component if exists, else 0.0)
  - Attach `JunctionTraversal { junction_node, turn_index, t: 0.0, lateral_offset, speed: current_speed }`
  - Do NOT advance route.current_step yet (that happens when junction traversal completes)
  - Return early (agent is now in junction traversal mode)
- If no junction data: proceed with existing instant edge transition
