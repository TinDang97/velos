<task type="auto">
  <name>Task 2: App.rs wiring -- ViewMode state, input routing, render dispatch, egui toggle</name>
  <files>crates/velos-gpu/src/app.rs, crates/velos-gpu/src/renderer3d.rs</files>
  <action>
Modify `GpuState` in app.rs to add 3D support. This is the largest integration task -- be precise about what changes.

**1. Add fields to GpuState:**
```rust
renderer_3d: Renderer3D,
orbit_camera: OrbitCamera,
view_mode: ViewMode,
view_transition: Option<ViewTransition>,
middle_pressed: bool,  // for middle-drag pan
last_cursor_pos: Option<(f32, f32)>,  // for orbit/pan delta
```

**2. In GpuState::new():**
- After creating Renderer, also create Renderer3D: `Renderer3D::new(&device, format, size.width, size.height)`
- Create OrbitCamera: `OrbitCamera::new(Vec2::new(size.width as f32, size.height as f32))`
- After uploading road lines to 2D renderer, also upload road geometry to Renderer3D: `renderer_3d.upload_road_geometry(&device, &sim.road_graph, &sim.junction_data)`
- Load meshes: `renderer_3d.load_meshes(&device)` (MeshSet)
- Set initial view_mode = ViewMode::TopDown2D

**3. In resize():**
- Also call `self.renderer_3d.resize(&self.device, new_size.width, new_size.height)`
- Also call `self.orbit_camera.resize(Vec2::new(...))`

**4. In update():**
- After existing 2D sim tick and instance upload...
- If view_mode is Perspective3D (or transitioning):
  - Update orbit camera uniform: `self.renderer_3d.update_camera(&self.queue, &self.orbit_camera)`
  - Update lighting: `self.renderer_3d.update_lighting(&self.queue, self.sim.elapsed_seconds())`
  - Build 3D instances: `let lod_buffers = self.sim.build_instances_3d(self.orbit_camera.eye_position())`
  - Upload: `self.renderer_3d.upload_agent_instances(&self.device, &self.queue, &lod_buffers)`
- If transitioning: advance ViewTransition.tick(dt), if complete set view_mode to target

**5. In render():**
- Branch on view_mode:
  - TopDown2D: existing render path (unchanged)
  - Perspective3D: call `self.renderer_3d.render_frame(encoder, view, ...)`
- During transition: render the target mode (no cross-fade for simplicity)

**6. Input handling in window_event():**
- Keyboard 'V' press: initiate view toggle
  - If TopDown2D -> start transition to Perspective3D, call `orbit_camera = OrbitCamera::from_camera_2d(&self.camera)`
  - If Perspective3D -> start transition to TopDown2D (map orbit_camera state back to camera_2d)
- Mouse input routing based on view_mode:
  - TopDown2D: existing pan logic (unchanged)
  - Perspective3D:
    - Left-drag: orbit (orbit_camera.orbit(dyaw, dpitch))
    - Scroll: zoom (orbit_camera.zoom_by(factor))
    - Middle-drag: pan (orbit_camera.pan(dx, dy))

**7. egui panel:**
- Add toggle button in the left control panel: "View: 2D / 3D" or "[V] Toggle 3D"
- Show current mode
- When clicked: same toggle logic as keyboard V

**8. Ensure app.rs stays under 700 lines.** If it exceeds, extract orbit camera input handling into a helper function or separate module. The current app.rs is 693 lines so this will likely push it over -- plan to extract input handling into a `fn handle_3d_input(&mut self, event: &WindowEvent)` method or a new `app_input.rs` module.

Note: Do NOT modify the existing 2D Renderer or Camera2D code. The 2D path must remain identical.
