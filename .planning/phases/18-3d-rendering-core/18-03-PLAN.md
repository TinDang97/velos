<objective>
Implement 3-tier LOD agent rendering (mesh/billboard/dot) with glTF mesh loading, time-of-day lighting system, and the 3D agent shaders.

Purpose: Agents are the core visual content. LOD ensures performance (most of 280K agents render as cheap dots, only nearby ones get 3D meshes). Lighting gives the scene temporal context and visual depth via diffuse shading on meshes.

Output: Complete agent rendering pipeline for 3D mode -- meshes for close range, billboards for mid, dots for far. Lighting uniforms that change with sim clock.
</objective>

1. Create `crates/velos-gpu/src/lighting.rs`:

   `LightingUniform` struct (#[repr(C)] Pod/Zeroable):
   - `sun_direction: [f32; 3]`, `_pad0: f32`
   - `sun_color: [f32; 3]`, `_pad1: f32`
   - `ambient_color: [f32; 3]`, `ambient_intensity: f32`
   Total 48 bytes.

   Keyframe presets (time_of_day in seconds, 86400 = full day):
   - Dawn (21600 = 6:00): sun_dir=(0.5, 0.3, 0.5) normalized, sun_color=(1.0, 0.7, 0.4), ambient=(0.8, 0.6, 0.4), intensity=0.3
   - Noon (43200 = 12:00): sun_dir=(0.0, -1.0, 0.2) normalized, sun_color=(1.0, 1.0, 0.95), ambient=(0.9, 0.9, 1.0), intensity=0.6
   - Sunset (64800 = 18:00): sun_dir=(-0.5, 0.3, -0.5) normalized, sun_color=(1.0, 0.5, 0.2), ambient=(0.8, 0.5, 0.3), intensity=0.25
   - Night (0 or 86400 = 0:00/24:00): sun_dir=(0.0, -1.0, 0.0), sun_color=(0.1, 0.1, 0.2), ambient=(0.2, 0.2, 0.4), intensity=0.15

   `compute_lighting(sim_elapsed_seconds: f64) -> LightingUniform`:
   - Map elapsed to time_of_day via modulo 86400
   - Find surrounding keyframes
   - Lerp all fields between keyframes
   - Normalize sun_direction after interpolation

2. Create `crates/velos-gpu/src/lod.rs`:

   `LodTier` enum: `Mesh`, `Billboard`, `Dot`.

   `classify_lod(distance: f32, previous_tier: Option<LodTier>) -> LodTier`:
   - Without previous (first frame): <50m=Mesh, <200m=Billboard, else=Dot
   - With previous (hysteresis): downgrade only at threshold * HYSTERESIS_FACTOR
     - Mesh -> Billboard: only if distance > 50 * 1.1 = 55
     - Billboard -> Dot: only if distance > 200 * 1.1 = 220
     - Upgrade (closer): at exact threshold (50m for Billboard->Mesh, 200m for Dot->Billboard)

   `LodBuffers` struct: holds per-vehicle-type, per-LOD-tier instance vectors:
   - `mesh_instances: HashMap<VehicleType, Vec<MeshInstance3D>>`
   - `billboard_instances: HashMap<VehicleType, Vec<BillboardInstance3D>>`
   - `dot_count: u32` (dots reuse existing 2D dot pipeline)
   - `classify_agents(positions: &[[f64;2]], vehicle_types: &[VehicleType], headings: &[f64], eye: Vec3) -> Self`
     - For each agent: compute distance to eye, classify LOD, build appropriate instance

   Billboard sizes by vehicle type (Claude's discretion):

   - Car: 2.0m x 1.5m
   - Bus: 3.0m x 3.5m
   - Truck: 2.5m x 3.0m
   - Pedestrian: 0.6m x 1.7m

   `Vertex3D` struct: `#[repr(C)] Pod/Zeroable`, `position: [f32;3]`, `normal: [f32;3]` (24 bytes).

   `LoadedMesh` struct: `vertices: Vec<Vertex3D>`, `indices: Vec<u32>`.

   `load_glb(path: &Path) -> Result<LoadedMesh>` per RESEARCH.md code example:
   - Use gltf::import(path)
   - Extract positions and normals from first primitive reader
   - Repack into Vertex3D vec
   - Extract indices as u32

   `generate_fallback_box(width: f32, height: f32, depth: f32) -> LoadedMesh`:
   - Procedural box mesh with normals (for when .glb files are missing)

   `MeshSet` struct: holds GPU buffers per vehicle type:
   - `HashMap<VehicleType, (Buffer, Buffer, u32)>` -- (vertex_buffer, index_buffer, index_count)
   - `load_all(device: &Device) -> Self`:
     - Try loading from `assets/models/{type}.glb` (motorbike.glb, car.glb, bus.glb, truck.glb, pedestrian.glb)
     - Fall back to procedural box if file not found
     - Upload to GPU buffers

4. Create `crates/velos-gpu/shaders/mesh_3d.wgsl` per RESEARCH.md shader code:
   - Group 0 binding 0: camera uniform (view_proj)
   - Group 0 binding 1: lighting uniform (sun_direction, sun_color, ambient_color, ambient_intensity)
   - Vertex input: position (loc 0), normal (loc 1)
   - Instance input: world_pos (loc 2), heading (loc 3), color (loc 4)
   - VS: rotate vertex around Y by heading, translate by world_pos, transform by view_proj
   - FS: diffuse + ambient shading: `lit = ambient_color * ambient_intensity * color + sun_color * max(dot(N, L), 0) * color`

5. Create `crates/velos-gpu/shaders/billboard_3d.wgsl`:
   - Same bind groups as mesh_3d
   - Instance input: world_pos (loc 0), size (loc 1), color (loc 2)
   - VS: expand 4 billboard corners from world_pos using camera-right and camera-up vectors (derived from view_proj inverse or passed as uniform)
   - Actually simpler approach: pass `camera_right: vec3<f32>` and `camera_up: vec3<f32>` as additional uniforms in camera bind group (extend the uniform buffer to 128 bytes: view_proj 64 + eye_pos 16 + camera_right 16 + camera_up 16)
   - FS: output instance color with ambient tint only (billboards are flat)

6. Extend `Renderer3D` in renderer3d.rs:
   - Add lighting uniform buffer + bind group (group 0, binding 1)
   - Extend camera uniform to include eye_position, camera_right, camera_up (for billboard shader)
   - Add mesh render pipeline (mesh_3d.wgsl + depth stencil)
   - Add billboard render pipeline (billboard_3d.wgsl + depth stencil)
   - Store MeshSet (loaded meshes)
   - Add `update_lighting(queue, sim_elapsed: f64)` -- compute_lighting -> write to uniform buffer
   - Add `upload_agent_instances(&mut self, device, queue, lod_buffers: &LodBuffers)`:
     - Upload mesh instance buffers per vehicle type
     - Upload billboard instance buffers per vehicle type
   - Extend `render_frame()` to draw agents after roads:
     - For each vehicle type with mesh instances: bind mesh vertex/index buffer, bind instance buffer, draw indexed instanced
     - For each vehicle type with billboard instances: bind billboard instance buffer, draw instanced (4 vertices per billboard, hardware quad)
     - Dot-tier agents: render through existing 2D dot pipeline or skip (they'll be handled in Plan 04 wiring)

7. Add naga validation tests for mesh_3d.wgsl and billboard_3d.wgsl.

8. Export new modules in lib.rs: `lighting`, `mesh_loader`, `lod`.
